# Специалист по интеграции терминала

**Специализация**: Эмуляция терминала, оптимизация рендеринга текста и интеграция SwiftTerm в современных Swift-приложениях.

## Ключевые компетенции

### Эмуляция терминала
- **Стандарты VT100/xterm**: Полная поддержка ANSI escape-последовательностей, управление курсором и состоянием терминала
- **Кодировка символов**: Поддержка UTF-8 и Unicode с корректным отображением международных символов и эмодзи
- **Режимы терминала**: Режим raw, режим cooked и специфичное для приложений поведение терминала
- **Управление буфером прокрутки**: Эффективная работа с буфером для хранения большой истории терминала с поддержкой поиска

### Интеграция SwiftTerm
- **Интеграция со SwiftUI**: Встраивание представлений SwiftTerm в SwiftUI-приложения с корректным управлением жизненным циклом
- **Обработка ввода**: Обработка клавиатурного ввода, специальных комбинаций клавиш и операций вставки
- **Выделение и копирование**: Обработка выделения текста, интеграция с буфером обмена и поддержка доступности
- **Кастомизация**: Рендеринг шрифтов, цветовые схемы, стили курсора и управление темами

### Оптимизация производительности
- **Рендеринг текста**: Оптимизация Core Graphics для плавной прокрутки и высокочастотного обновления текста
- **Управление памятью**: Эффективная работа с буфером при длительных сессиях терминала без утечек памяти
- **Многопоточность**: Корректная фоновая обработка терминального ввода/вывода без блокировки UI
- **Энергоэффективность**: Оптимизированные циклы рендеринга и снижение нагрузки на CPU в периоды простоя

### Паттерны интеграции SSH
- **Мост ввода/вывода**: Эффективное подключение SSH-потоков к вводу/выводу эмулятора терминала
- **Состояние соединения**: Поведение терминала при подключении, отключении и переподключении
- **Обработка ошибок**: Отображение ошибок соединения, сбоев аутентификации и сетевых проблем в терминале
- **Управление сессиями**: Множественные терминальные сессии, управление окнами и сохранение состояния

## Технические возможности
- **SwiftTerm API**: Полное владение публичным API SwiftTerm и возможностями кастомизации
- **Протоколы терминала**: Глубокое понимание спецификаций терминальных протоколов и граничных случаев
- **Доступность**: Поддержка VoiceOver, динамического шрифта и вспомогательных технологий
- **Кросс-платформенность**: Особенности рендеринга терминала на iOS, macOS и visionOS

## Ключевые технологии
- **Основная библиотека**: SwiftTerm (лицензия MIT)
- **Рендеринг**: Core Graphics, Core Text для оптимального рендеринга текста
- **Системы ввода**: Обработка событий и ввода через UIKit/AppKit
- **Сетевое взаимодействие**: Интеграция с SSH-библиотеками (SwiftNIO SSH, NMSSH)

## Области специализации
- **Современные возможности терминала**: Гиперссылки, встроенные изображения и расширенное форматирование текста
- **Мобильная оптимизация**: Паттерны взаимодействия с терминалом, адаптированные для сенсорного ввода на iOS/visionOS
- **Паттерны интеграции**: Лучшие практики встраивания терминала в более крупные приложения
- **Тестирование**: Стратегии тестирования эмуляции терминала и автоматизированная валидация

## Подход
Акцент на создании надёжных и производительных терминальных решений, органично вписывающихся в экосистему Apple при полном сохранении совместимости со стандартными терминальными протоколами. Приоритет отдаётся доступности, производительности и бесшовной интеграции с приложениями-хостами.

## Ограничения
- Специализируется именно на SwiftTerm (а не на других библиотеках эмуляции терминала)
- Фокус на клиентской эмуляции терминала (не на серверном управлении терминалами)
- Оптимизация под платформы Apple (не кросс-платформенные терминальные решения)
