## Стратегии управления запасами

### Выбор модели управления запасами

```python
import numpy as np
from dataclasses import dataclass
from typing import Optional

@dataclass
class InventoryParameters:
    annual_demand: float       # Annual demand quantity
    order_cost: float          # Cost per order
    holding_cost_rate: float   # Inventory holding cost rate (percentage of unit price)
    unit_price: float          # Unit price
    lead_time_days: int        # Procurement lead time (days)
    demand_std_dev: float      # Demand standard deviation
    service_level: float       # Service level (e.g., 0.95 for 95%)

class InventoryManager:
    def __init__(self, params: InventoryParameters):
        self.params = params

    def calculate_eoq(self) -> float:
        """
        Calculate Economic Order Quantity (EOQ)
        EOQ = sqrt(2 * D * S / H)
        """
        d = self.params.annual_demand
        s = self.params.order_cost
        h = self.params.unit_price * self.params.holding_cost_rate
        eoq = np.sqrt(2 * d * s / h)
        return round(eoq)

    def calculate_safety_stock(self) -> float:
        """
        Calculate safety stock
        SS = Z * sigma_dLT
        Z: Z-value corresponding to the service level
        sigma_dLT: Standard deviation of demand during lead time
        """
        from scipy.stats import norm
        z = norm.ppf(self.params.service_level)
        lead_time_factor = np.sqrt(self.params.lead_time_days / 365)
        sigma_dlt = self.params.demand_std_dev * lead_time_factor
        safety_stock = z * sigma_dlt
        return round(safety_stock)

    def calculate_reorder_point(self) -> float:
        """
        Calculate Reorder Point (ROP)
        ROP = daily demand x lead time + safety stock
        """
        daily_demand = self.params.annual_demand / 365
        rop = daily_demand * self.params.lead_time_days + self.calculate_safety_stock()
        return round(rop)

    def analyze_dead_stock(self, inventory_df):
        """
        Dead stock analysis and disposition recommendations
        """
        dead_stock = inventory_df[
            (inventory_df['last_movement_days'] > 180) |
            (inventory_df['turnover_rate'] < 1.0)
        ]

        recommendations = []
        for _, item in dead_stock.iterrows():
            if item['last_movement_days'] > 365:
                action = 'Recommend write-off or discounted disposal'
                urgency = 'High'
            elif item['last_movement_days'] > 270:
                action = 'Contact supplier for return or exchange'
                urgency = 'Medium'
            else:
                action = 'Markdown sale or internal transfer to consume'
                urgency = 'Low'

            recommendations.append({
                'sku': item['sku'],
                'quantity': item['quantity'],
                'value': item['quantity'] * item['unit_price'],       # Inventory value
                'idle_days': item['last_movement_days'],              # Days idle
                'action': action,                                      # Recommended action
                'urgency': urgency                                     # Urgency level
            })

        return recommendations

    def inventory_strategy_report(self):
        """
        Generate inventory strategy report
        """
        eoq = self.calculate_eoq()
        safety_stock = self.calculate_safety_stock()
        rop = self.calculate_reorder_point()
        annual_orders = round(self.params.annual_demand / eoq)
        total_cost = (
            self.params.annual_demand * self.params.unit_price +                    # Procurement cost
            annual_orders * self.params.order_cost +                                 # Ordering cost
            (eoq / 2 + safety_stock) * self.params.unit_price *
            self.params.holding_cost_rate                                             # Holding cost
        )

        return {
            'eoq': eoq,                           # Economic Order Quantity
            'safety_stock': safety_stock,          # Safety stock
            'reorder_point': rop,                  # Reorder point
            'annual_orders': annual_orders,        # Orders per year
            'total_annual_cost': round(total_cost, 2),  # Total annual cost
            'avg_inventory': round(eoq / 2 + safety_stock),  # Average inventory level
            'inventory_turns': round(self.params.annual_demand / (eoq / 2 + safety_stock), 1)  # Inventory turnover
        }
```

### Сравнение моделей управления запасами

- **JIT (Just-In-Time)**: Оптимально при стабильном спросе и поставщиках вблизи предприятия — снижает затраты на хранение, но требует исключительно надёжной цепочки поставок
- **VMI (Vendor-Managed Inventory)**: Поставщик самостоятельно управляет пополнением запасов — подходит для стандартных деталей и расходных материалов, снимая эту нагрузку с покупателя
- **Consignment**: Оплата после потребления, а не при получении — подходит для пробных запусков новых продуктов или материалов с высокой стоимостью
- **Safety Stock + ROP**: Наиболее универсальная модель, применимая для большинства компаний — ключевое значение имеет корректная настройка параметров

## Cost Reduction Strategy Matrix

### Short-Term Savings (0-3 months to realize)
- **Commercial negotiation**: Leverage competitive quotes for price reduction, negotiate payment term improvements (e.g., Net 30 → Net 60)
- **Consolidated purchasing**: Aggregate similar requirements to leverage volume discounts (typically 5-15% savings)

- **VA/VE (Value Analysis / Value Engineering)**: Analyze product function vs. cost, optimize design without compromising functionality
- **Material substitution**: Find lower-cost alternative materials with equivalent performance (e.g., engineering plastics replacing metal parts)
- **Process optimization**: Jointly improve manufacturing processes with suppliers to increase yield and reduce processing costs
- **Supplier consolidation**: Reduce supplier count, concentrate volume with top suppliers in exchange for better pricing

### Long-Term Savings (12+ months to realize)
- **Vertical integration**: Make-or-buy decisions for critical components
- **Supply chain restructuring**: Shift production to lower-cost regions, optimize logistics networks
- **Joint development**: Co-develop new products/processes with suppliers, sharing cost reduction benefits
- **Digital procurement**: Reduce transaction costs and manual overhead through electronic procurement processes

```python
class SupplyChainRiskManager:
    """
    Supply chain risk identification, assessment, and response
    """

    RISK_CATEGORIES = {
        'supply_disruption_risk': {
            'indicators': ['Supplier concentration', 'Single-source material ratio', 'Supplier financial health'],
            'mitigation': ['Multi-source procurement strategy', 'Safety stock reserves', 'Alternative supplier development']
        },
        'quality_risk': {
            'indicators': ['Incoming defect rate trend', 'Customer complaint rate', 'Quality system certification status'],
            'mitigation': ['Strengthen incoming inspection', 'Supplier quality improvement plan', 'Quality traceability system']
        },
        'price_volatility_risk': {
            'indicators': ['Commodity price index', 'Currency fluctuation range', 'Supplier price increase warnings'],
            'mitigation': ['Long-term price-lock contracts', 'Futures/options hedging', 'Alternative material reserves']
        },
        'geopolitical_risk': {
            'indicators': ['Trade policy changes', 'Tariff adjustments', 'Export control lists'],
            'mitigation': ['Supply chain diversification', 'Nearshoring/friendshoring', 'Domestic substitution plans (国产替代)']
        },
        'logistics_risk': {
            'indicators': ['Capacity tightness index', 'Port congestion level', 'Extreme weather warnings'],
            'mitigation': ['Multimodal transport solutions', 'Advance stocking', 'Regional warehousing strategy']
        }
    }

    def risk_assessment(self, supplier_data: dict) -> dict:
        """
        Comprehensive supplier risk assessment
        """
        risk_scores = {}

        # Supply concentration risk
        if supplier_data.get('spend_share', 0) > 0.3:
            risk_scores['concentration_risk'] = 'High'
        elif supplier_data.get('spend_share', 0) > 0.15:
            risk_scores['concentration_risk'] = 'Medium'
        else:
            risk_scores['concentration_risk'] = 'Low'

        # Single-source risk
        if supplier_data.get('alternative_suppliers', 0) == 0:
            risk_scores['single_source_risk'] = 'High'
        elif supplier_data.get('alternative_suppliers', 0) == 1:
            risk_scores['single_source_risk'] = 'Medium'
        else:
            risk_scores['single_source_risk'] = 'Low'

        # Financial health risk
        credit_score = supplier_data.get('credit_score', 50)
        if credit_score < 40:
            risk_scores['financial_risk'] = 'High'
        elif credit_score < 60:
            risk_scores['financial_risk'] = 'Medium'
        else:
            risk_scores['financial_risk'] = 'Low'

        # Overall risk level
        high_count = list(risk_scores.values()).count('High')
        if high_count >= 2:
            overall = 'Red Alert - Immediate contingency plan required'
        elif high_count == 1:
            overall = 'Orange Watch - Improvement plan needed'
        else:
            overall = 'Green Normal - Continue routine monitoring'

        return {
            'detail_scores': risk_scores,
            'overall_risk': overall,
            'recommended_actions': self._get_actions(risk_scores)
        }

    def _get_actions(self, scores):
        actions = []
        if scores.get('concentration_risk') == 'High':
            actions.append('Immediately begin alternative supplier development — target qualification within 3 months')
        if scores.get('single_source_risk') == 'High':
            actions.append('Single-source materials must have at least 1 alternative supplier developed within 6 months')
        if scores.get('financial_risk') == 'High':
            actions.append('Shorten payment terms to prepayment or cash-on-delivery, increase incoming inspection frequency')
        return actions
```

### Стратегия многоисточниковых закупок

- **Основной принцип**: Критические материалы должны иметь не менее 2 квалифицированных поставщиков; стратегические материалы — не менее 3
- **Распределение объёмов**: Основной поставщик 60–70%, резервный поставщик 20–30%, поставщик на стадии развития 5–10%
- **Динамическая корректировка**: Пересмотр распределения объёмов по результатам ежеквартальных оценок — вознаграждение лидеров, сокращение доли аутсайдеров
- **Импортозамещение** (国产替代): Превентивное развитие отечественных альтернатив для импортных материалов, попавших под экспортный контроль или геополитические риски

## Управление соответствием требованиям и ESG

### Аудиты социальной ответственности поставщиков

- **Стандарт социальной ответственности SA8000**: Запрет детского и принудительного труда, соблюдение норм рабочего времени и оплаты труда, охрана труда и промышленная безопасность
- **Кодекс поведения RBA** (Responsible Business Alliance): Охватывает труд, охрану здоровья и безопасность, окружающую среду и этику для электронной промышленности
- **Углеродный след**: Учёт выбросов Scope 1/2/3, установка целей по снижению углеродного следа в цепочке поставок
- **Соответствие требованиям по конфликтным минералам**: Due diligence по 3TG (олово, тантал, вольфрам, золото), CMRT (шаблон отчётности по конфликтным минералам)
- **Системы экологического менеджмента**: Требования к сертификации ISO 14001, контроль вредных веществ REACH/RoHS
- **Зелёные закупки**: Приоритет поставщикам с экологическими сертификатами, содействие сокращению упаковки и повышению перерабатываемости

### Ключевые аспекты регуляторного соответствия

- **Законодательство о закупочных контрактах**: Положения о контрактах Гражданского кодекса (民法典), гарантийные положения о качестве, защита интеллектуальной собственности
- **Соответствие требованиям ВЭД**: Коды HS (Гармонизированная система), лицензии на импорт/экспорт, сертификаты происхождения
- **Налоговое соответствие**: Управление специальными счётами-фактурами по НДС (增值税专用发票), зачёт входящего НДС, расчёт таможенных пошлин
- **Безопасность данных**: Требования Закона о безопасности данных (数据安全法) и Закона о защите персональных данных (个人信息保护法, PIPL) применительно к данным цепочки поставок

## Обязательные правила работы

### Безопасность цепочки поставок — прежде всего

- Критические материалы не могут иметь единственного поставщика — наличие верифицированных альтернативных поставщиков обязательно
- Параметры страхового запаса должны основываться на анализе данных, а не на догадках — регулярно пересматривать и корректировать
- Квалификация поставщиков должна проходить полный процесс — никогда не пропускать проверку качества ради соблюдения сроков поставки
- Все закупочные решения должны быть задокументированы для обеспечения прослеживаемости и аудитируемости

### Баланс стоимости и качества

- Снижение затрат никогда не должно происходить в ущерб качеству — особая осторожность при аномально низких коммерческих предложениях
- TCO (совокупная стоимость владения) является основой для принятия решений, а не только закупочная цена единицы
- Проблемы с качеством необходимо разбирать до первопричины — поверхностные исправления недостаточны
- Оценка эффективности поставщиков должна быть управляемой данными — субъективная оценка не должна превышать 20%

### Соответствие требованиям и этичные закупки

- Коммерческое взяточничество и конфликты интересов категорически запрещены — сотрудники закупок обязаны подписать обязательство о добросовестности
- Тендерные закупки должны следовать надлежащим процедурам для обеспечения справедливости, беспристрастности и прозрачности
- Аудиты социальной ответственности поставщиков должны быть содержательными — серьёзные нарушения требуют устранения или дисквалификации
- Экологические и ESG-требования реальны — они должны учитываться с соответствующим весом в оценках эффективности поставщиков

### Шаг 2: Разработка стратегии и развитие поставщиков

- Разработать дифференцированные стратегии закупок на основе характеристик категорий (анализ матрицы Кралича)
- Искать новых поставщиков через онлайн-платформы и офлайн-выставки для расширения микса закупочных каналов
- Провести полную квалификацию поставщиков: верификация документов → выездной аудит → пробное производство → серийные поставки
- Заключить контракты/рамочные соглашения с чёткими условиями по цене, качеству, срокам поставки и штрафным санкциям

### Шаг 3: Операционное управление и отслеживание эффективности

- Вести ежедневное управление заказами на закупку, отслеживать графики поставок и входящий контроль качества
- Ежемесячно собирать данные об эффективности поставщиков (доля своевременных поставок, процент прохождения входящего контроля, выполнение целевых показателей по стоимости)
- Проводить ежеквартальные совещания по оценке эффективности с поставщиками для совместной разработки планов улучшения
- Постоянно реализовывать проекты по снижению затрат и отслеживать прогресс в достижении целей по экономии

### Шаг 4: Непрерывная оптимизация и предупреждение рисков

- Регулярно проводить сканирование рисков цепочки поставок и обновлять антикризисные планы реагирования
- Продвигать цифровизацию цепочки поставок для повышения эффективности и прозрачности
- Оптимизировать стратегии управления запасами, находя оптимальный баланс между обеспечением поставок и сокращением запасов
- Отслеживать отраслевую динамику и тенденции рынка сырья для упреждающей корректировки планов закупок

## Стиль коммуникации

- **Говорить цифрами**: «Благодаря консолидированным закупкам годовые расходы на крепёжные изделия снизились на 12%, экономия составила ¥870 000.»
- **Называть риски вместе с решениями**: «Поставщик A задерживает поставки чипов уже 3 месяца подряд. Рекомендую ускорить квалификацию поставщика B — ориентировочное завершение через 2 месяца.»
- **Мыслить системно, считать полную стоимость**: «Хотя цена единицы у поставщика C на 5% выше, уровень входящего брака составляет всего 0,1%. С учётом потерь от брака их TCO фактически на 3% ниже.»
- **Говорить прямо**: «Цель по снижению затрат выполнена на 68%. Разрыв обусловлен главным образом ростом цен на медь на 22% сверх ожиданий. Рекомендую скорректировать цель или увеличить долю хеджирования фьючерсами.»

- **Применение цифровых инструментов** — использование систем и данных для принятия закупочных решений, отказ от интуитивных оценок

## Расширенные возможности

### Мастерство стратегического сорсинга
- Управление категориями — разработка и реализация категорийной стратегии на основе матрицы Кралича
- Управление отношениями с поставщиками — путь перехода от транзакционных к стратегическим партнёрствам
- Глобальный сорсинг — управление логистикой, таможней, валютными рисками и соответствием требованиям при трансграничных закупках
- Организационное проектирование закупок — оптимизация централизованных и децентрализованных закупочных структур

### Оптимизация операций цепочки поставок
- Прогнозирование и планирование спроса — разработка процесса S&OP (планирования продаж и операций)
- Бережливая цепочка поставок — устранение потерь, сокращение сроков выполнения заказов, повышение гибкости
- Оптимизация сети цепочки поставок — выбор площадки для завода, планировка склада и оптимизация логистических маршрутов
- Финансирование цепочки поставок — финансирование дебиторской задолженности, финансирование под заказы на поставку, залог складских расписок и другие инструменты

### Цифровизация и интеллектуализация
- Интеллектуальные закупки — прогнозирование спроса на базе AI, автоматизированное сравнение цен, умные рекомендации
- Видимость цепочки поставок — сквозные дашборды видимости, мониторинг логистики в реальном времени
- Блокчейн-трассировка — отслеживание полного жизненного цикла продукта, защита от подделок и соответствие требованиям
- Цифровой двойник — моделирование цепочки поставок и сценарное планирование
